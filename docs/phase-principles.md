# Phase Execution Principles

## Philosophy

Architecture maturity must follow product maturity.

Do NOT introduce complexity before it is required.

---

# Current Founder Priorities

1. Shipping speed
2. Maintainability
3. Simplicity
4. Customer validation
5. Long-term scalability

---

# Incremental Layering

Build in phases:

1. Foundation
2. TypeScript shells
3. Framework setup
4. Database
5. AI integration
6. External integrations
7. Deployments
8. Productization

---

# Complexity Rules

Avoid introducing early:
- microservices
- CQRS
- event sourcing
- Kubernetes
- workflow engines
- AI agent orchestration
- complex abstractions

---

# Codex / Claude Rules

Always:
- preserve readability
- prefer incremental changes
- explain reasoning briefly
- avoid massive rewrites
- maintain architectural consistency

---

# Startup Rule

Useful systems > impressive systems.

# Frontend App Router Architecture

```
app/
  layout.tsx      — root layout (runs on every route)
                    sets <html lang> and <body>
                    imports globals.css (Tailwind entry point)
                    exports Metadata (title, description)
  page.tsx        — root route handler (GET /)
                    Server Component by default
                    renders dashboard placeholder UI
  globals.css     — Tailwind v4 CSS entry: @import "tailwindcss"
```

Future routes follow the filesystem convention:
```
app/
  dashboard/
    page.tsx         # GET /dashboard
  restaurants/
    page.tsx         # GET /restaurants
    [id]/
      page.tsx       # GET /restaurants/:id
```

# Tailwind v4 Setup

Tailwind v4 uses a PostCSS plugin (`@tailwindcss/postcss`) and requires only one CSS file entry:

```css
/* app/globals.css */
@import "tailwindcss";
```

No `tailwind.config.ts` required for default configuration.
Custom theme tokens added via `@theme` directive in CSS when needed.

# Known Environment Warning

`next build` and `next dev` emit: "Found lockfile missing swc dependencies, patching..."

Root cause: Next.js tries to patch `pnpm-lock.yaml` to explicitly list SWC native binaries, but
cannot reach `localhost:4873` (local Verdaccio registry configured globally on this machine).
SWC IS installed and operational — `@next+swc-linux-x64-gnu@16.2.6` is present in pnpm store.
Warning is non-fatal. Resolved when the machine's global registry config is corrected.
